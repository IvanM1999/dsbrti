Guia Definitivo: Programando sua Primeira Prensa no TIA Portal

Bem-vindo! Se você é totalmente leigo e nunca mexeu no software da Siemens, não se preocupe. Este guia foi feito para pegar na sua mão e mostrar clique por clique como transformar a lógica da nossa apostila em um programa de verdade.

Passo 1: O que a máquina precisa fazer? (Entendendo o problema)

Antes de clicar em qualquer lugar, precisamos saber o que vai entrar de sinal no nosso CLP e o que vai sair. Pense no CLP como o cérebro da máquina.

  • Sinais que entram (Entradas / Inputs): Botões que o operador aperta e sensores que detectam coisas.
  • Sinais que saem (Saídas / Outputs): O que faz a máquina se mexer, neste caso, a válvula que empurra o cilindro da prensa para baixo.

A prensa só pode descer se o operador apertar o botão E a grade estiver fechada. Ela deve subir se bater no fim de curso, se a grade abrir ou se apertarem a emergência.

Passo 2: Dando "Apelidos" aos Fios (Criando as PLC Tags)

O CLP não entende nomes como "Botão" ou "Sensor". Ele entende endereços elétricos como %I0.1. Para não termos que decorar esses códigos, nós criamos as "Tags" (etiquetas), que são apelidos amigáveis.

  1. No menu do lado esquerdo do TIA Portal (chamado Project tree), procure pela pasta do seu CLP (geralmente tem um ícone verde escrito PLC_1).
  2. Clique na setinha ao lado dele para expandir as opções.
  3. Abra a pasta PLC tags e dê um clique duplo em Show all tags (Mostrar todas as tags).
  4. Vai abrir uma tabela vazia no centro da tela. Preencha exatamente assim:
Name (O apelido) Data Type (O tipo) Address (O pino do CLP) Para que serve?
Botao_Emergencia Bool (Liga/Desliga) %I0.1 Botão vermelho de parar tudo.
Botao_S3_Iniciar Bool %I0.3 Botão verde que o operador aperta para descer a prensa.
Sensor_B1_Grade Bool %I0.4 Avisa se a grade de proteção está fechada.
Sensor_B2_Cilindro Bool %I0.5 Avisa que a prensa chegou lá embaixo (fim de curso).
Valvula_M0_Prensa Bool %Q0.0 (Note que é Q, de Saída) O atuador que empurra a prensa pra baixo.

Passo 3: Criando a "Folha de Papel" para programar (Criando uma FC)

Nunca devemos jogar tudo de qualquer jeito no bloco principal do CLP. Vamos criar uma "pastinha" (Função) só para a lógica da prensa.

  1. Volte no menu do lado esquerdo (Project tree).
  2. Expanda a pasta Program blocks (Blocos de programa).
  3. Dê um clique duplo em Add new block (Adicionar novo bloco).
  4. Uma janela vai pular na tela. Clique no botão azul escrito FC (Function).
  5. Na caixinha "Name", digite Controle_Da_Prensa.
  6. Confirme se em "Language" está escrito LAD (Linguagem Ladder).
  7. Clique em OK. Uma tela em branco com uma linha verde (Network 1) vai aparecer.

Passo 4: Desenhando a Lógica (O Código Ladder)

A linguagem Ladder parece um esquema elétrico. A energia vem da esquerda e precisa chegar até a direita para ligar a prensa. Vamos colocar os obstáculos (contatos) no caminho.

Dica de Ouro:
- Contato Aberto [ ]: Só deixa a energia passar se o sensor/botão for ativado.
- Contato Fechado [/]: Deixa a energia passar o tempo todo, e CORTA a energia se o sensor for ativado.
- Bobina ( ): É a máquina que queremos ligar. Fica sempre no final, na direita.

Desenhando a Linha:

  1. Olhe para a parte superior da tela. Você verá símbolos de [ ], [/] e ( ).
  2. Clique no símbolo do Contato Aberto [ ] e arraste para a linha verde. Digite em cima dele: Botao_S3_Iniciar e dê Enter.
  3. Arraste outro Contato Aberto [ ] e coloque do lado direito dele. Digite: Sensor_B1_Grade. (Isso obriga a grade estar fechada).
  4. Arraste um Contato Fechado [/] e coloque do lado direito. Digite: Sensor_B2_Cilindro. (Isso deixa a energia passar até a prensa bater no fundo. Quando bater, o contato abre e desliga tudo).
  5. Arraste um Contato Aberto [ ] e coloque do lado direito. Digite: Botao_Emergencia. (Como o botão físico já é do tipo fechado na fábrica, no software usamos o aberto. Se baterem na emergência na vida real, o sinal some e a energia do software corta).
  6. Por fim, arraste uma Bobina ( ) para o final da linha. Digite: Valvula_M0_Prensa.

Fazendo o "Selo" (Para o operador não precisar ficar segurando o botão):

  1. Clique na linha verde bem ANTES do Botao_S3_Iniciar.
  2. Na barra superior, clique no botão Open Branch (uma setinha preta apontando para baixo). Uma perninha vai nascer para baixo.
  3. Arraste um Contato Aberto [ ] para essa linha de baixo. Digite o nome da própria saída: Valvula_M0_Prensa.
  4. Clique no final dessa linha de baixo, e depois clique no botão Close Branch (uma setinha apontando para cima). Feche a linha conectando logo APÓS o Botao_S3_Iniciar.
  5. Pronto! Agora, se a prensa ligar, a energia passa por baixo, pelo próprio contato dela, e o operador pode soltar o botão S3.

Passo 5: Avisando o Cérebro Principal (Chamando a FC no Main)

Se você não fizer isso, o CLP vai ignorar tudo o que você fez! O CLP só lê automaticamente um bloco chamado Main [OB1].

  1. No menu esquerdo, dentro de Program blocks, dê um duplo clique no bloco verde Main [OB1].
  2. Ele vai abrir vazio.
  3. Vá novamente no menu esquerdo, clique em cima da sua FC Controle_Da_Prensa (não dê duplo clique, apenas segure o clique).
  4. Arraste-a para dentro da linha verde (Network 1) do Main [OB1] e solte.
  5. Ela vai aparecer como uma caixinha quadrada. Agora o CLP sabe que precisa executar a sua lógica da prensa!

Passo 6: Mandar para a Máquina e Testar

  1. Na barra lá em cima, clique no ícone de Compile (um papelzinho com uns números 0 e 1 e uma setinha). O TIA Portal vai verificar se você não digitou nenhum nome errado. A barrinha de baixo tem que ficar verde.
  2. Clique no ícone Download to device (um computadorzinho com uma seta azul apontando para um CLP).
  3. Uma janela vai abrir para procurar o CLP na rede. Siga clicando em Load (Carregar) até terminar.
  4. Para ver o programa funcionando na tela do computador, clique no ícone do óculos (Monitoring on/off) lá em cima. Tudo o que tiver energia elétrica passando vai ficar com a linha verde brilhante!
